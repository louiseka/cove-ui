## When to use

### Primary Button

Use a primary button for the main action on a page or section. For example:

- There is one clear action you want users to take.
- Progressing to the next step.
- Confirming the primary choice.

### Secondary Button

Use a secondary button for supporting or alternative actions. For example:

- There is more than one action, but only one is primary.
- Cancel, go back, or secondary workflows.
- Less important actions alongside a primary button.

### Disabled Button

Use a disabled button when an action is temporarily unavailable. For example:

- Required fields are incomplete.
- User must complete a step first.
- Action is not yet permitted.

### Loading Button

Use a loading button when an action is in progress after user interaction. For example:

- Submitting a form.
- Processing a request.
- Saving data.

### Submit Button

Use a submit button specifically for form submission. For example:

- Sending form data to a server.
- Completing a multi-step form.
- Finalising user input.

---

## Accessibility

Buttons should be used for actions, not navigation. If the element takes a user to another page, use a link instead.

<div className="docsAccessibility">

### Semantic usage

- Use a `<button>` for actions such as submitting forms, opening dialogs, or triggering changes in the interface.
- Use a link (`<a>`) for navigation between pages or external destinations.

### Button text

Button text should clearly describe the action it performs. For example:

- “Save and continue”
- “Sign in”
- “Add another item”
- “Submit form”

Avoid vague labels such as “Click here” or “Submit” without context.

</div>
